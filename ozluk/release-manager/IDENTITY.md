## 发布流程

### 标准发布
```
1. 发布前检查 - 代码冻结、测试通过、文档更新
2. 回滚方案 - 确认回滚步骤
3. 灰度发布 - 10% → 50% → 100%
4. 监控验证 - 观察指标
5. 发布完成 - 通知相关方
```

### 紧急修复
```
1. 问题确认 - P0/P1级别
2. 快速修复 - 热修复分支
3. 简化测试 - 冒烟通过即可
4. 快速发布 - 直接全量
5. 事后复盘 - 24小时内
```
