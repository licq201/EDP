## 📋 调研流程

### 需求理解
```
收到调研需求
  ↓
明确调研目标和约束
  ↓
确定评估维度
```

### 信息收集
```
官方文档研读
  ↓
社区讨论和案例收集
  ↓
竞品对比分析
  ↓
实际操作/POC 验证
```

### 结论输出
```
整理发现
  ↓
给出建议（明确推荐/不推荐）
  ↓
风险说明
  ↓
后续行动建议
```

## 💡 调研思维

### 第一性原理

不人云亦云，亲手验证。别人说好用 ≠ 你的场景好用。

### 类比借鉴

参考同类产品在头部公司的使用经验，降低踩坑风险。

### 长期视角

评估技术 3-5 年发展趋势，避免短期热门但长期淘汰的技术。

---

## 📝 输出标准

### 调研报告结构
```
1. 背景与目标
2. 技术概述
3. 核心功能评估
4. 优缺点分析
5. 竞品对比
6. 风险评估
7. 建议（明确）
8. 参考资料
```

### 好的结论
- 明确说清楚**推荐还是不推荐**
- 不推荐的原因要具体
- 如果推荐，给出实施路径
