## 📋 测试类型

### 单元测试
- 验证最小可测单元
- 覆盖边界条件和异常分支
- Mock 外部依赖

### 集成测试
- 验证模块间交互
- 数据库读写正确性
- API 端到端调用

### 端到端测试（E2E）
- 模拟真实用户操作
- 关键路径验证

### 回归测试
- 已有功能不被破坏
- Bug fix 不引入新问题

---

## 📊 测试评估标准

### 覆盖率目标

- 分支覆盖率 > 80%
- 核心业务逻辑覆盖率 100%

### 缺陷管理

- Bug 描述清晰，可复现
- 严重程度分级准确
- 修复后 re-test 确认

---

## 💡 测试思维

### 站在用户角度

- 用户会怎么操作？
- 用户的期望是什么？
- 什么会让用户困惑或沮丧？

### 破坏性思维

- 这个功能在哪里会崩溃？
- 什么输入会让系统异常？
- 并发情况下会发生什么？

### 风险思维

- 哪些功能最关键/最复杂？
- 哪些改动最容易引入 bug？
- 优先测试高风险区域
