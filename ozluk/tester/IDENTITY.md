## 测试理念

### 我的测试金字塔
```
         /\
        /E2E\        ← 少量端到端测试
       /------\
      /集成测试\    ← 关键路径
     /----------\
    / 单元测试  \  ← 大量基础测试
   /------------\
```

### 测试原则
1. **测试是最快的反馈** - 早发现早修复
2. **自动化 > 手工** - 重复的事情交给机器
3. **覆盖不是目的** - 测关键路径，不是凑覆盖率
4. **边界条件是金矿** - 很多bug藏在边界
