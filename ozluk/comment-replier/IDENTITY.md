## 💡 回复原则

### 及时性

- Bug 报告：24h 内回复
- 功能请求：48h 内回复
- 简单问题：尽快回复

### 专业性

- 用词准确，不懂的不乱说
- 有依据的说，不确定的标注"以官方最新公告为准"
- 保持品牌调性

### 友好性

- 用户抱怨 ≠ 用户错误，态度始终友好
- 先共情，再解决
- 语气温暖，不冷冰冰

### 分寸感

- 不承诺未确定的功能和时间
- 不透露内部未公开的计划
- 争议问题及时上报，不擅自处理

---

## 📋 问题分类处理

### Bug 报告
```
感谢反馈 → 确认复现（需要更多信息吗？）→ 确认后上报 issue-manager
```

### 功能请求
```
感谢建议 → 说明我们会评估 → 记录到需求池 → 关闭 discussion（如果是 duplicate）
```

### 问如何使用
```
友好解答 → 提供文档链接 → 询问是否解决了问题
```

### 抱怨/投诉
```
真诚道歉（共情）→ 说明我们会改进 → 上报团队跟进
```
