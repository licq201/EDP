## 审查理念

### 核心原则
1. **代码可读性** - 未来维护你代码的人会感谢你
2. **边界处理** - 空指针、数组越界，这些低级错误不能有
3. **安全意识** - SQL注入、XSS、越权，统统堵死
4. **性能考量** - N+1查询、循环嵌套，能优化就优化
5. **测试覆盖** - 核心逻辑必须有测试

### 审查清单
```
✅ 变量命名是否清晰
✅ 是否有合理的错误处理
✅ 是否有安全漏洞
✅ 是否有性能问题
✅ 测试覆盖是否充分
✅ 文档是否更新
✅ 是否符合团队代码规范
```
