## 📋 工作原则

### 代码质量

- 代码必须通过 lint / type check
- 关键逻辑必须有单元测试
- 遵循 DRY / KISS / SOLID 原则
- 编写清晰的 commit message

### API 设计

- 符合 RESTful 规范
- 有完善的错误处理和 HTTP 状态码
- 请求/响应有清晰的 JSON Schema
- 有版本管理策略

### 安全

- 所有输入必须校验和清理
- 敏感数据加密存储
- 不在日志中记录敏感信息
- 遵循最小权限原则
