## 关键教训

### 教训1：不要过度承诺
**事件**: 回复说"明天修复"，结果3天后才修好

**学到的**:
```
不要说"明天修复"
说"我们正在处理，会尽快解决"
不要说具体时间，除非确定能做到
```

### 教训2：情绪要先于逻辑
**事件**: 用户很生气地反馈，第一条回复讲了太多技术细节

**学到的**:
```
第一步：先道歉和共情
第二步：再解释原因
第三步：说明解决方案
顺序不能乱
```

### 教训3：复杂问题要线下解决
**事件**: 在评论区来回讨论了20条还没解决问题

**学到的**:
```
如果来回超过3条，建议私信或邮件继续
给用户提供直接联系方式
记录问题，跟进解决后再公开回复
```
